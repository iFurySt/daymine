# 产品判断原则

## 核心用户

当前核心用户是自己和同类 power user：愿意本地部署、愿意让 AI Agent 长期维护资料库、每天只想快速检查结果的人。未来可以扩展到小团队，但早期不为多人权限和 SaaS 运营优化。

## 真正价值

- 把每天散落的信息整理成长期可回溯的个人资料库。
- 让 Agent 能安全、稳定地在本地 workspace 中执行任务并留下产物。
- 用 dashboard 把“今天该看什么、最近发生了什么、历史怎么追溯”组织清楚。
- 数据属于用户，能被 Obsidian、编辑器、Git 和备份工具直接读取。

## 取舍顺序

1. 可回溯性优先于短期炫酷展示。
2. 本地可运行优先于云端功能完整度。
3. 文件格式可读优先于数据库查询性能；性能问题出现后再引入可替换索引。
4. Agent Provider 抽象优先于单一厂商深度绑定。
5. 复刻 tididi 的成熟视觉风格优先于重新设计一套 UI。

## 早期判断标准

- 是否能一个命令启动。
- 是否无需登录即可看到真实 dashboard。
- 是否能把 Agent 运行和产物追踪清楚。
- 是否能用 Markdown/JSON 解释和修复数据。
- 是否每个新增能力都有测试或可重复验证路径。

## 成熟期判断标准

- 多个 Agent Provider 能共享同一执行模型。
- Panel 扩展不需要修改核心渲染器也能覆盖常见信息形态。
- 长期运行后 workspace 仍然可读、可迁移、可备份。
- 失败可诊断，重要操作有 run history 和日志摘要。
